## 🔍 Описание проблемы

### Исходная проблема
Пользователь сообщил о проблеме, касающейся **только платформы macOS**:
- ❌ Не даёт войти в приложение по PIN-коду
- ❌ Не устанавливает PIN-код
- ✅ На Windows всё работает корректно

### Логи ошибки
```
flutter: [AuthLocalDataSource] verifyPin: hash = NULL
flutter: [AuthLocalDataSource] verifyPin: salt = NULL
flutter: [AuthLocalDataSource] verifyPin: PIN НЕ УСТАНОВЛЕН (hash или salt = null)
flutter: [AuthLocalDataSource] isPinSetup: ОШИБКА = SqfliteFfiException(sqlite_error: 1, 
  SqliteException(1): while preparing statement, no such table: auth_data
```

### Корневая причина
Таблица `auth_data` **отсутствовала** в базе данных на macOS:
- База данных была создана с версией схемы 1 (без таблицы `auth_data`)
- При обновлении приложения миграция не применялась автоматически
- Флаг миграции из SharedPreferences уже был установлен, поэтому миграция пропускалась

### 2. Исправление проблемы

**Файл: `lib/main.dart`**

Добавлена проверка существования таблицы `auth_data` при запуске приложения:

```dart
// Проверяем существование таблицы auth_data
debugPrint('[MAIN] Проверка существования таблицы auth_data...');
final tables = await db.rawQuery(
  "SELECT name FROM sqlite_master WHERE type='table' AND name='auth_data'",
);

if (tables.isEmpty) {
  debugPrint('[MAIN] Таблица auth_data не найдена, создаём...');
  await db.execute('''
    CREATE TABLE auth_data (
      key TEXT PRIMARY KEY,
      value TEXT NOT NULL,
      created_at INTEGER NOT NULL
    )
  ''');
  debugPrint('[MAIN] Таблица auth_data создана');
} else {
  debugPrint('[MAIN] Таблица auth_data существует');
}
```

## 🚀 Рекомендации к развёртыванию

### Для пользователей macOS

1. **Обновите приложение:**
   ```bash
   git pull origin main
   flutter build macos
   ```

2. **При первом запуске:**
   - Таблица `auth_data` будет создана автоматически
   - PIN-код можно будет установить и использовать

3. **Проверьте работу:**
   - Установите PIN-код (4-8 цифр)
   - Выйдите из приложения
   - Войдите по PIN-коду

### Для разработчиков

1. **Мониторинг логов:**
   ```bash
   flutter run | grep AuthLocalDataSource
   ```

2. **Проверка пути к БД:**
   ```
   [DatabaseHelper] Путь к базе данных: ...
   ```

3. **Диагностика проблем:**
   - Ищите логи с префиксом `[AuthLocalDataSource]`
   - Проверяйте наличие таблицы `auth_data`
