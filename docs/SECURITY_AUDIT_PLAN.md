## 🎯 Критические области проверки

### 1. Аутентификация и авторизация

**Область:** `lib/data/datasources/auth_local_datasource.dart`

**Проверки:**
- [ ] **PIN хранение**
  - [ ] PIN хранится только в SQLite (не в SharedPreferences)
  - [ ] Хэш PIN деривируется через PBKDF2 с солью
  - [ ] Соль генерируется через CSPRNG
  - [ ] Длина PIN: 4-8 цифр

- [ ] **Защита от подбора**
  - [ ] Блокировка после 5 неудачных попыток
  - [ ] Длительность блокировки: 30 секунд
  - [ ] Счётчик попыток хранится в SQLite
  - [ ] Блокировка сбрасывается после успешного входа

- [ ] **Хранение ключей**
  - [ ] Ключи затираются после использования (`CryptoUtils.secureWipeKey`)
  - [ ] Ключи не сохраняются в логах
  - [ ] Ключи не передаются между изолятами

### 2. Шифрование данных

**Область:** `lib/data/datasources/encryptor_local_datasource.dart`

**Проверки:**
- [ ] **Алгоритм шифрования**
  - [ ] Используется ChaCha20-Poly1305 (AEAD)
  - [ ] Nonce уникален для каждого сообщения
  - [ ] Длина nonce: 12 байт для ChaCha20
  - [ ] MAC (authentication tag) проверяется при расшифровке

- [ ] **Деривация ключа**
  - [ ] PBKDF2 с HMAC-SHA256
  - [ ] Итерации: 10000 (минимум)
  - [ ] Соль уникальна для каждого пользователя
  - [ ] Длина ключа: 256 бит

- [ ] **Генерация случайных чисел**
  - [ ] Используется `Random.secure()` (CSPRNG)
  - [ ] Соль: 32 байта
  - [ ] Nonce: 12 байт
  - [ ] Векторы инициализации уникальны

### 3. Хранение данных

**Область:** `lib/data/database/`

**Проверки:**
- [ ] **SQLite база данных**
  - [ ] Файл БД хранится в защищённой директории
  - [ ] Чувствительные данные зашифрованы перед записью
  - [ ] Нет plaintext паролей в БД
  - [ ] Индексы не содержат чувствительных данных

- [ ] **Миграции БД**
  - [ ] Миграции применяют изменения атомарно
  - [ ] Откат при ошибке миграции
  - [ ] Проверка версии схемы перед миграцией

- [ ] **Таблица auth_data**
  - [ ] `key` TEXT PRIMARY KEY
  - [ ] `value` TEXT NOT NULL (зашифрованные данные)
  - [ ] Нет триггеров для экспорта данных

### 4. Работа с памятью

**Область:** `lib/core/utils/crypto_utils.dart`

**Проверки:**
- [ ] **Затирание чувствительных данных**
  - [ ] `secureWipeKey()` затирает ключи после использования
  - [ ] `secureWipeData()` затирает расшифрованные данные
  - [ ] Затирание происходит в `finally` блоке
  - [ ] Обработка ошибок затирания

- [ ] **Иммутабельность**
  - [ ] Ключи объявлены как `final`
  - [ ] Чувствительные данные не копируются
  - [ ] Использование `Uint8List` для бинарных данных

### 5. Логи и отладка

**Область:** Все файлы с `debugPrint`

**Проверки:**
- [ ] **Отсутствие чувствительных данных в логах**
  - [ ] Нет паролей в логах
  - [ ] Нет ключей шифрования в логах
  - [ ] Нет PIN-кодов в логах
  - [ ] Нет зашифрованных данных в логах

- [ ] **Production vs Debug**
  - [ ] `debugPrint` отключён в production
  - [ ] Нет `print()` в production коде
  - [ ] Логи не сохраняются в файл

### 6. Импорт/Экспорт

**Область:** `lib/data/formats/passgen_format.dart`, `lib/data/datasources/storage_local_datasource.dart`

**Проверки:**
- [ ] **Формат .passgen**
  - [ ] HEADER: "PASSGEN_V1" (10 байт)
  - [ ] VERSION: 1 (1 байт)
  - [ ] FLAGS: 0 (1 байт)
  - [ ] METADATA_LENGTH: 2 байта
  - [ ] METADATA: JSON с EncryptionMetadata
  - [ ] PBKDF2_NONCE: 32 байта
  - [ ] CHACHA_NONCE: 12 байт (исправлено в 0.5.2)
  - [ ] DATA_LENGTH: 4 байта
  - [ ] DATA: зашифрованный JSON
  - [ ] MAC: 16 байт

- [ ] **Валидация импорта**
  - [ ] Проверка HEADER
  - [ ] Проверка VERSION
  - [ ] Проверка MAC перед расшифровкой
  - [ ] Обработка повреждённых файлов
  - [ ] Проверка на дубликаты (service + login)

- [ ] **Экспорт данных**
  - [ ] Данные шифруются перед записью
  - [ ] Файл сохраняется во временной директории
  - [ ] Пользователь выбирает место сохранения
  - [ ] Файл удаляется после экспорта

### 7. Буфер обмена

**Область:** `lib/presentation/widgets/copyable_password.dart`

**Проверки:**
- [ ] **Автоочистка буфера**
  - [ ] Таймер: 60 секунд

## 🧪 План тестирования на уязвимости

### Этап 1: Статический анализ (1-2 часа)

**Инструменты:**
```bash
# Статический анализ Dart
flutter analyze

# Проверка зависимостей
dart pub outdated
dart pub upgrade --dry-run

# Поиск уязвимостей в коде
grep -r "print(" lib/
grep -r "SharedPreferences" lib/
grep -r "password" lib/ --include="*.dart"
```

**Чек-лист:**
- [ ] Нет `print()` с чувствительными данными
- [ ] Нет `SharedPreferences` для хранения паролей/PIN
- [ ] Все зависимости актуальны
- [ ] Нет предупреждений `flutter analyze`

### Этап 2: Динамический анализ (2-3 часа)

**Тесты на аутентификацию:**
```bash
# 1. Подбор PIN (брутфорс)
- Ввести 5 неправильных PIN
- Проверить блокировку на 30 секунд
- Ввести правильный PIN после блокировки

# 2. Обход аутентификации
- Попытка входа без установки PIN
- Попытка входа с пустым PIN
- Попытка входа с PIN < 4 символов
- Попытка входа с PIN > 8 символов
- Попытка входа с не-цифровым PIN
```

**Тесты на шифрование:**
```bash
# 1. Проверка уникальности nonce
- Зашифровать одно сообщение 100 раз
- Проверить, что все nonce уникальны

# 2. Проверка MAC
- Повредить зашифрованные данные (1 бит)
- Попытаться расшифровать
- Ожидать: ошибка аутентификации

# 3. Проверка деривации ключа
- Ввести один PIN 10 раз
- Проверить, что хэш одинаковый
- Ввести другой PIN
- Проверить, что хэш отличается
```

**Тесты на хранение данных:**
```bash
# 1. Проверка SQLite
- Найти файл БД на устройстве
- Открыть через DB Browser for SQLite
- Проверить, что пароли зашифрованы
- Проверить, что PIN hash зашифрован

# 2. Проверка логов
- Включить логирование
- Выполнить все операции
- Проверить логи на чувствительные данные
```

### Этап 3: Пентест (3-4 часа)

**Атака 1: Извлечение данных из БД**
```
1. Получить root/jailbreak доступ
2. Скопировать файл БД
3. Попытаться расшифровать без PIN
4. Ожидать: данные зашифрованы
```

**Атака 2: Перехват буфера обмена**
```
1. Установить менеджер буфера обмена
2. Скопировать пароль из приложения
3. Подождать 60 секунд
4. Проверить буфер обмена
5. Ожидать: буфер очищен
```

**Атака 3: Memory dump**
```
1. Запустить приложение
2. Войти с PIN
3. Сделать дамп памяти процесса
4. Поискать PIN в памяти
5. Ожидать: PIN затёрт
```

**Атака 4: Side-channel (timing attack)**
```
1. Измерить время проверки неправильного PIN
2. Измерить время проверки правильного PIN
3. Сравнить времена
4. Ожидать: времена одинаковые (constant-time сравнение)
```

### Этап 4: Аудит зависимостей (1 час)

**Проверка зависимостей:**
```bash
# Анализ зависимостей
dart pub deps
dart pub outdated

# Проверка на известные уязвимости
# (использовать https://snyk.io/advisor/flutter или аналоги)
```

**Чек-лист:**
- [ ] `cryptography` — нет CVE
- [ ] `sqflite` — нет CVE
- [ ] `flutter_lints` — нет CVE
- [ ] Все зависимости из pub.dev
- [ ] Нет forked зависимостей

### Этап 5: Отчётность (1-2 часа)

**Структура отчёта:**
1. Резюме (Executive Summary)
2. Найденные уязвимости (с критичностью)
3. Доказательства концепции (PoC)
4. Рекомендации по исправлению
5. План исправлений

**Шкала критичности:**
- **Critical** — немедленное исправление
- **High** — исправить в течение 7 дней
- **Medium** — исправить в течение 30 дней
- **Low** — исправить в следующем релизе
- **Info** — рекомендация
