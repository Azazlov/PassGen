## 2. НАЗНАЧЕНИЕ И ЦЕЛИ РАЗРАБОТКИ

### 2.1 Назначение
Приложение предназначено для:
*   Генерации криптографически стойких паролей по настраиваемым параметрам.
*   Безопасного локального хранения учётных данных (логин, пароль, сервис).
*   Шифрования конфиденциальных данных перед записью в базу данных.
*   Экспорта и импорта данных для переноса между устройствами.

### 2.2 Цели дипломного проекта
1.  **Практическая:** Создание рабочего кроссплатформенного приложения.
2.  **Исследовательская:** Анализ эффективности алгоритма ChaCha20-Poly1305 на мобильных устройствах.
3.  **Научная новизна:** Реализация механизма шифрования конфигураций генерации паролей в составе общей структуры данных.

## 3. ТРЕБОВАНИЯ К ФУНКЦИОНАЛУ

### 3.1 Аутентификация и Безопасность

| Функция | Описание |
|---|---|
| **Вход по PIN-коду** | Обязательная процедура при запуске. Длина PIN: 4–8 цифр. |
| **Деривация ключа** | Использование KDF (PBKDF2/Argon2) для преобразования PIN в ключ шифрования. |
| **Хранение ключей** | Мастер-ключ хранится только в оперативной памяти (RAM) во время сессии. |
| **Блокировка** | Автоматическая блокировка при неактивности более 5 минут. |
| **Защита от подбора** | Временная блокировка ввода (30 сек) после 5 неудачных попыток. |
| **Биометрия** | Не реализуется (допускается как перспектива развития). |

### 3.2 Генератор паролей

| Параметр | Диапазон / Значения |
|---|---|
| **Длина пароля** | 8–64 символа (настраиваемый диапазон) |
| **Наборы символов** | Строчные (a-z), Заглавные (A-Z), Цифры (0-9), Спецсимволы (!@#$%) |
| **Уникальность** | Опция «Без повторяющихся символов» |
| **Исключения** | Опция «Исключить похожие символы» (l, 1, I, O, 0) |
| **Пресеты** | «Стандартный», «Надёжный», «Максимальный», «PIN-код», «Пользовательский» |
| **Оценка стойкости** | Визуальный индикатор надёжности (zxcvbn + эвристика) |

### 3.3 Хранилище данных

| Функция | Описание |
|---|---|
| **CRUD записей** | Создание, чтение, обновление, удаление записей о паролях |
| **Категоризация** | Системные категории (по умолчанию) + пользовательские категории |
| **Поиск** | Фильтрация по названию сервиса и категории |
| **Копирование** | Копирование пароля в буфер обмена с таймаутом очистки |
| **История** | Ведение журнала изменений (таблица `security_logs`) |

### 3.4 Импорт и Экспорт

| Формат | Статус | Описание |
|---|---|---|
| **JSON (Miniified)** | ✅ Обязательно | Все данные в одной зашифрованной строке (Base64) |
| **PassGen (.passgen)** | ✅ Обязательно | Фирменный формат файла с зашифрованным содержимым |
| **CSV** | 🔲 Перспектива | Текстовый формат без шифрования (только экспорт) |

### 3.5 Логирование событий

**Таблица логируемых событий:**

| Код события | Описание | Данные |
|---|---|---|
| `AUTH_SUCCESS` | Успешный вход | Timestamp |
| `AUTH_FAILURE` | Неудачная попытка | Timestamp, попытка № |
| `PWD_CREATED` | Создание пароля | ID записи, категория |
| `PWD_ACCESSED` | Просмотр пароля | ID записи |
| `PWD_DELETED` | Удаление пароля | ID записи |
| `DATA_EXPORT` | Экспорт данных | Кол-во записей |
| `DATA_IMPORT` | Импорт данных | Результат (успех/ошибка) |
| `SETTINGS_CHG` | Изменение настроек | Тип настройки |

*Хранение логов: последние 1000 записей, старые удаляются автоматически.*

## 4. ТРЕБОВАНИЯ К БАЗЕ ДАННЫХ (SQLite)

### 4.1 Схема данных (5 таблиц)

**Таблица 1: `categories`**
| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PRIMARY KEY | Уникальный ID |
| `name` | TEXT NOT NULL | Название категории |
| `icon` | TEXT | Иконка (emoji/код) |
| `is_system` | INTEGER DEFAULT 0 | Флаг системной категории |
| `created_at` | INTEGER | Timestamp создания |

**Таблица 2: `password_entries`**
| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PRIMARY KEY | Уникальный ID |
| `category_id` | INTEGER FK → categories.id | Ссылка на категорию |
| `service` | TEXT NOT NULL | Название сервиса |
| `login` | TEXT | Логин/Email |
| `encrypted_password` | BLOB NOT NULL | Зашифрованный пароль |
| `nonce` | BLOB | Nonce для шифрования |
| `created_at` | INTEGER | Timestamp создания |
| `updated_at` | INTEGER | Timestamp обновления |

**Таблица 3: `password_configs`**
| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PRIMARY KEY | Уникальный ID |
| `entry_id` | INTEGER FK → password_entries.id | Ссылка на запись |
| `strength` | INTEGER | Уровень сложности |
| `min_length` | INTEGER | Мин. длина |
| `max_length` | INTEGER | Макс. длина |
| `flags` | INTEGER | Битовая маска категорий символов |
| `require_unique` | INTEGER | Требовать уникальность |
| `encrypted_config` | BLOB | Зашифрованные доп. настройки |

**Таблица 4: `security_logs`**
| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PRIMARY KEY | Уникальный ID |
| `action_type` | TEXT NOT NULL | Тип события |
| `timestamp` | INTEGER NOT NULL | Время события |
| `details` | TEXT | JSON с деталями (без чувствительных данных) |

**Таблица 5: `app_settings`**
| Поле | Тип | Описание |
|---|---|---|
| `key` | TEXT PRIMARY KEY | Ключ настройки |
| `value` | TEXT NOT NULL | Значение настройки |
| `encrypted` | INTEGER DEFAULT 0 | Флаг шифрования значения |

### 4.2 Связи между таблицами
*   `password_entries.category_id` → `categories.id` (Many-to-One)
*   `password_configs.entry_id` → `password_entries.id` (One-to-One)
*   `security_logs` — независимая таблица событий
*   `app_settings` — независимая таблица настроек

## 5. ТРЕБОВАНИЯ К АРХИТЕКТУРЕ ПО

### 5.1 Clean Architecture (3 слоя)

| Слой | Модуль (lib/) | Ответственность |
|---|---|---|
| **Presentation** | `presentation/` | UI, Widgets, Controllers (State Management) |
| **Domain** | `domain/` | Entities, Use Cases, Repository Interfaces |
| **Data** | `data/` | Repository Implementations, Data Sources, SQLite |
| **Core** | `core/` | Utils, Constants, Errors (общие утилиты) |
| **App** | `app/` | DI Container, Routing, Theme |

### 5.2 Основные зависимости
*   **State Management:** Provider (`ChangeNotifier`)
*   **Database:** `sqflite` или `drift` (SQLite)
*   **Cryptography:** `cryptography` (ChaCha20-Poly1305)
*   **UI Kit:** Material 3, Google Fonts
*   **Utilities:** `shared_preferences` (для некритичных настроек)

### 5.3 Диаграмма потоков данных
```
Пользователь → UI (Screen) → Controller → Use Case → Repository → DataSource → SQLite/Crypto
```

## 7. ТРЕБОВАНИЯ К НАДЁЖНОСТИ И БЕЗОПАСНОСТИ

| Аспект | Требование |
|---|---|
| **Шифрование данных** | ChaCha20-Poly1305 (AES-GCM альтернатива) |
| **Хранение мастер-пароля** | ❌ Запрещено. Хранится только хеш PIN для проверки |
| **Ключи в памяти** | Очищаются при закрытии приложения или таймауте |
| **Буфер обмена** | Автоочистка через 60 секунд после копирования пароля |
| **Резервное копирование** | Обязательная функция экспорта перед критическими изменениями |
| **Целостность данных** | Проверка поли-тега (Poly1305) при дешифровании |
