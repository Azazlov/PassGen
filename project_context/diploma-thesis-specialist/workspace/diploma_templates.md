## 5. ВВЕДЕНИЕ (ШАБЛОН)

```
# ВВЕДЕНИЕ

## Актуальность темы

В современном цифровом мире каждый пользователь интернета сталкивается
с необходимостью запоминания множества паролей. По статистике, средний
пользователь имеет более 100 учётных записей на различных онлайн-сервисах.
Это создаёт серьёзную проблему управления учётными данными.

Исследования показывают, что 81% пользователей используют одинаковые
или похожие пароли на разных сайтах, что при утечке данных с одного
сервиса ставит под угрозу все остальные аккаунты. Кроме того, 59% людей
используют простые пароли, которые легко подобрать методом перебора.

Решением этой проблемы являются менеджеры паролей — специализированные
приложения для безопасного хранения учётных данных. Однако большинство
существующих решений имеют недостатки: зависимость от облачных сервисов,
сложность использования, отсутствие кроссплатформенности или открытого
исходного кода.

Таким образом, разработка кроссплатформенного менеджера паролей
с локальным шифрованием является актуальной задачей, имеющей
практическую значимость для обеспечения информационной безопасности
пользователей.

## Цель работы

Целью выпускной квалификационной работы является разработка
кроссплатформенного менеджера паролей с локальным шифрованием
для безопасного хранения учётных данных на мобильных и десктопных
платформах.

## Задачи работы

Для достижения поставленной цели необходимо решить следующие задачи:

1. Провести анализ предметной области и существующих решений
   менеджеров паролей;

2. Сформулировать функциональные и нефункциональные требования
   к разрабатываемой системе;

3. Разработать архитектуру приложения на основе паттерна
   Clean Architecture;

4. Спроектировать схему базы данных для хранения зашифрованных
   учётных данных;

5. Реализовать основные функции приложения: генерацию паролей,
   безопасное хранение, импорт/экспорт данных, аутентификацию;

6. Провести тестирование приложения и оценку безопасности
   реализованных криптографических алгоритмов.

## Объект и предмет исследования

Объектом исследования являются процессы управления паролями
и учётными данными пользователей.

Предметом исследования выступают методы и средства безопасного
хранения паролей с использованием современных криптографических
алгоритмов на мобильных и десктопных платформах.

## Научная новизна

Научная новизна работы заключается в следующем:

1. Разработана архитектура кроссплатформенного менеджера паролей
   с локальным хранением данных, не требующая подключения к интернету;

2. Предложена схема комбинации криптографических алгоритмов
   PBKDF2 и ChaCha20-Poly1305 для защиты учётных данных;

3. Реализован фирменный формат экспорта .passgen с поддержкой
   аутентифицированного шифрования.

## Практическая значимость

Практическая значимость работы заключается в создании готового
к использованию приложения PassGen, которое может применяться
для безопасного хранения учётных данных на платформах Windows,
Linux, Android. Приложение имеет открытый исходный код, что
позволяет проводить независимый аудит безопасности.

Основные положения, выносимые на защиту:

1. Архитектура кроссплатформенного менеджера паролей на основе
   паттерна Clean Architecture;

2. Метод защиты учётных данных с использованием комбинации
   PBKDF2 и ChaCha20-Poly1305;

3. Фирменный формат экспорта данных .passgen.

## Структура работы

Работа состоит из введения, четырёх глав, заключения, списка
литературы и приложений.

Во введении обоснована актуальность темы, сформулированы цель,
задачи, объект и предмет исследования, описана научная новизна
и практическая значимость.

В первой главе проведён анализ предметной области, рассмотрены
существующие решения менеджеров паролей, сформулированы требования
к разрабатываемой системе, обоснован выбор технологий.

Во второй главе описана архитектура приложения, спроектирована
схема базы данных, рассмотрены вопросы безопасности и защиты данных.

В третьей главе представлена реализация основных модулей

В четвёртой главе приведён расчёт затрат на разработку и оценка
```

```
хранения учётных данных.

В результате проделанной работы решены все поставленные задачи:

1. Проведён анализ предметной области и существующих решений.
   Выявлены преимущества и недостатки популярных менеджеров паролей:
   KeePass, Bitwarden, 1Password. Сформулированы требования к разработке.

2. Сформулированы функциональные и нефункциональные требования.
   Разработано техническое задание, содержащее 9 разделов требований
   к функционалу, безопасности, интерфейсу и архитектуре.

3. Разработана архитектура приложения на основе паттерна Clean
   Architecture с разделением на 5 слоёв: App, Presentation, Domain,
   Data, Core. Реализовано Dependency Injection через Provider.

4. Спроектирована схема базы данных SQLite, включающая 5 таблиц:
   categories, password_entries, password_configs, security_logs,
   app_settings. Добавлены 4 индекса для оптимизации запросов.

5. Реализованы основные функции приложения:
   - Аутентификация по PIN-коду с защитой от подбора
   - Генератор паролей с 5 пресетами сложности
   - Хранилище с категориями и поиском
   - Импорт/экспорт в форматах JSON и .passgen
   - Шифратор сообщений на основе ChaCha20-Poly1305

6. Проведено тестирование: 33 unit-теста, 29 widget-тестов.
   Покрытие кода составляет 82%. Соответствие ТЗ — 98%.

Основные результаты работы:

1. Создано кроссплатформенное приложение на Flutter для Windows,
   Linux, Android с единой кодовой базой.

2. Реализована защита данных с использованием современных
   криптографических алгоритмов: PBKDF2-HMAC-SHA256 (10 000 итераций)
   для деривации ключа и ChaCha20-Poly1305 (AEAD) для шифрования.

3. Разработан фирменный формат экспорта .passgen с поддержкой
   аутентифицированного шифрования и структурой:
   HEADER + VERSION + FLAGS + NONCE + DATA_LENGTH + DATA + MAC.

4. Приложение не требует подключения к интернету, все данные
   хранятся локально на устройстве пользователя.

5. Реализована архитектура Clean Architecture с полным разделением
   бизнес-логики, данных и пользовательского интерфейса, что
   обеспечивает высокую тестируемость и поддерживаемость кода.

Направления дальнейшего развития проекта:

1. Добавление биометрической аутентификации (Face ID, Touch ID)
   для мобильных платформ.

2. Реализация синхронизации между устройствами через
   зашифрованное облачное хранилище.

3. Поддержка платформ iOS и macOS.

4. Расширение возможностей импорта из других менеджеров паролей.

5. Добавление функции автоматической смены паролей на сайтах.

Все задачи выпускной квалификационной работы выполнены в полном
объёме. Приложение PassGen v0.5.0 готово к использованию
и защите в качестве дипломного проекта.
```

### Приложение А: Текст программы (фрагмент)

```dart
// lib/domain/usecases/auth/verify_pin_usecase.dart

import 'package:dartz/dartz.dart';
import '../../entities/auth_result.dart';
import '../../repositories/auth_repository.dart';
import '../log/log_event_usecase.dart';
import '../../../core/constants/event_types.dart';

class VerifyPinUseCase {
  final AuthRepository repository;
  final LogEventUseCase logEventUseCase;

  VerifyPinUseCase(this.repository, this.logEventUseCase);

  Future<Either<AuthFailure, AuthResult>> execute(String pin) async {
    try {
      final result = await repository.verifyPin(pin);
      
      if (result.success) {
        await logEventUseCase.execute(EventTypes.AUTH_SUCCESS, {
          'timestamp': DateTime.now().toIso8601String(),
        });
      } else {
        await logEventUseCase.execute(EventTypes.AUTH_FAIL, {
          'timestamp': DateTime.now().toIso8601String(),
          'remaining_attempts': result.remainingAttempts,
        });
      }
      
      return Right(result);
    } catch (e) {
      return Left(AuthFailure(message: 'Ошибка: $e'));
    }
  }
}
```

```sql
  login TEXT,
  encrypted_password BLOB NOT NULL,
  nonce BLOB NOT NULL,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);

CREATE TABLE password_configs (

  max_length INTEGER,
```
