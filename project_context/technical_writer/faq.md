## Общие вопросы

### ❓ Что такое PassGen?

**PassGen** — это кроссплатформенное приложение для генерации, хранения и управления паролями с использованием современных криптографических методов. Все данные хранятся локально на вашем устройстве.

### ❓ Какие платформы поддерживаются?

- ✅ Windows (10/11)
- ✅ Linux (Ubuntu, Debian, Fedora)
- ✅ Android (8.0+)
- ⏳ macOS (в разработке)
- ⏳ iOS (в разработке)

### ❓ Это бесплатно?

Да, PassGen полностью бесплатен и распространяется под лицензией MIT. Исходный код доступен на [GitHub](https://github.com/azazlov/passgen).

### ❓ Где хранятся мои данные?

Все данные хранятся **локально** на вашем устройстве в зашифрованной SQLite базе данных. Никакие данные не передаются на внешние серверы.

**Пути к данным:**
- **Windows:** `%APPDATA%/pass_gen/`
- **Linux:** `~/.local/share/pass_gen/`
- **Android:** `/data/data/com.example.pass_gen/databases/`

### ❓ Могу ли я синхронизировать данные между устройствами?

В текущей версии автоматическая синхронизация не поддерживается. Вы можете вручную экспортировать данные в файл `.passgen` и импортировать их на другом устройстве.

## Безопасность

### ❓ Насколько безопасен PassGen?

PassGen использует современные криптографические алгоритмы:

| Компонент | Технология |
|-----------|------------|
| Шифрование | ChaCha20-Poly1305 (AEAD) |
| Деривация ключа | PBKDF2-HMAC-SHA256, 10 000 итераций |
| Генерация случайных чисел | CSPRNG (Cryptographically Secure PRNG) |
| Аутентификация | Poly1305 MAC tag |

### ❓ Что будет, если я забуду PIN-код?

К сожалению, **восстановление PIN-кода невозможно**. Это сделано из соображений безопасности. При потере PIN-кода потребуется переустановка приложения с потерей данных.

**Рекомендация:** Запишите PIN-код и храните в безопасном месте.

### ❓ Можно ли обойти защиту?

Приложение разработано с учётом современных требований безопасности:
- PIN-код хэшируется с использованием PBKDF2
- После 5 неудачных попыток — блокировка на 30 секунд
- Данные шифруются перед записью в базу
- Исходный код открыт для аудита

### ❓ Как часто нужно менять пароль?

Рекомендуется менять пароли:
- **Критические аккаунты** (банки, почта): каждые 3-6 месяцев
- **Социальные сети**: каждые 6-12 месяцев
- **Второстепенные сервисы**: по необходимости

PassGen ведёт журнал всех изменений паролей.

### ❓ Что такое оценка надёжности пароля?

Приложение оценивает пароли по шкале от 0 до 4:

| Оценка | Описание | Рекомендация |
|--------|----------|--------------|
| 0 🔴 | Очень слабый | Не использовать |
| 1 🟠 | Слабый | Только для неважных аккаунтов |
| 2 🟡 | Средний | Минимально допустимый |
| 3 🟢 | Надёжный | Рекомендуется |
| 4 🟣 | Очень надёжный | Для критических аккаунтов |

Оценка вычисляется с использованием библиотеки `zxcvbn` и эвристических правил.

## Аутентификация

### ❓ Как установить PIN-код?

При первом запуске приложение автоматически предложит установить PIN-код. Также это можно сделать в настройках.

**Требования к PIN:**
- Длина: 4-8 цифр
- Только цифры (0-9)

### ❓ Как сменить PIN-код?

1. Перейдите в **Настройки**
2. Нажмите **"Сменить PIN"**
3. Введите текущий PIN
4. Введите новый PIN
5. Подтвердите новый PIN

### ❓ Как удалить PIN-код?

1. Перейдите в **Настройки**
2. Нажмите **"Удалить PIN"**
3. Введите текущий PIN для подтверждения

> ⚠️ **Внимание:** После удаления PIN доступ к приложению будет возможен без аутентификации!

### ❓ Почему приложение блокируется через 5 минут?

Это функция безопасности для защиты от несанкционированного доступа. Таймер неактивности сбрасывается при любом взаимодействии с приложением.

### ❓ Что делать после блокировки?

Просто введите ваш PIN-код для разблокировки. Все данные сохраняются.

### ❓ Можно ли использовать биометрию?

В текущей версии поддерживается только PIN-код. Биометрическая аутентификация (отпечаток, Face ID) запланирована в будущих версиях.

## Генератор паролей

### ❓ Как сгенерировать пароль?

1. Перейдите на вкладку **Генератор**
2. Выберите категорию
3. Настройте параметры (длина, сложность)
4. Нажмите **"Сгенерировать пароль"**

### ❓ Какая максимальная длина пароля?

Максимальная длина пароля — **64 символа**. Минимальная — **8 символов**.

### ❓ Что означают уровни сложности?

| Уровень | Символы | Примерная длина |
|---------|---------|-----------------|
| 1 Стандартный | a-z, A-Z | 8-10 |
| 2 Средний | a-z, A-Z, 0-9 | 10-12 |
| 3 Высокий | a-z, A-Z, 0-9, спецсимволы | 12-16 |
| 4 Очень высокий | Все наборы | 16-24 |
| 5 Максимальный | Все наборы | 24-32 |

### ❓ Можно ли исключить похожие символы?

Да, в версии 0.5.0 реализована функция исключения похожих символов. При включении этой опции исключаются символы: `l`, `1`, `I`, `O`, `0`.

### ❓ Можно ли генерировать пароли без повторяющихся символов?

Да, в версии 0.5.0 реализована функция генерации паролей без повторяющихся символов. Каждый символ будет использоваться только один раз.

### ❓ Как сохранить сгенерированный пароль?

1. После генерации введите название сервиса
2. При необходимости укажите логин
3. Выберите категорию
4. Нажмите **"Сохранить"**

Пароль будет зашифрован и сохранён в базе данных.

### ❓ Что такое категория пароля?

Категория помогает организовать пароли по группам. Доступны 7 системных категорий:
- 🌐 Интернет
- 📱 Социальные сети
- 🎮 Развлечения
- 💼 Работа
- 💰 Финансы
- 🛒 Покупки
- 📧 Почта

Также можно создавать пользовательские категории.

## Хранилище

### ❓ Как посмотреть сохранённые пароли?

Перейдите на вкладку **Хранилище**. Используйте навигацию для просмотра записей.

### ❓ Как найти конкретный пароль?

**Поиск:**
1. Введите название сервиса в поле поиска
2. Результаты фильтруются автоматически

**Фильтр по категории:**
1. Нажмите на иконку категории
2. Выберите нужную категорию

### ❓ Как скопировать пароль?

Нажмите на поле с паролем — он автоматически скопируется в буфер обмена. Появится уведомление об успешном копировании.

**Автоочистка буфера (v0.5.0):** пароль автоматически удаляется из буфера через 60 секунд после копирования.

### ❓ Как удалить пароль?

1. Выберите запись
2. Нажмите кнопку **"Удалить"**
3. Подтвердите действие

> ⚠️ **Внимание:** Удаление необратимо!

### ❓ Можно ли редактировать пароль?

В текущей версии прямое редактирование не поддерживается. Удалите запись и создайте новую.

### ❓ Сколько паролей можно хранить?

Ограничений нет. Однако при очень большом количестве записей (тысячи) производительность может снизиться.

## Импорт и экспорт

### ❓ Зачем нужно резервное копирование?

Резервное копирование защищает от потери данных при:
- Сбое устройства
- Переустановке приложения
- Переходе на новое устройство

### ❓ Какие форматы экспорта поддерживаются?

| Формат | Описание | Шифрование |
|--------|----------|------------|
| **JSON** | Текстовый формат | ❌ Нет |
| **.passgen** | Фирменный формат | ✅ ChaCha20-Poly1305 |

### ❓ Какой формат выбрать?

- **JSON** — для быстрого просмотра и редактирования
- **.passgen** — для безопасного хранения и передачи

### ❓ Как экспортировать данные?

1. Перейдите в **Хранилище**
2. Нажмите **"Экспорт"**
3. Выберите формат
4. Для .passgen введите мастер-пароль
5. Сохраните файл

### ❓ Как импортировать данные?

1. Перейдите в **Хранилище**
2. Нажмите **"Импорт"**
3. Выберите формат
4. Укажите файл
5. Для .passgen введите мастер-пароль
6. Подтвердите импорт

### ❓ Что будет при импорте дубликатов?

Дубликаты определяются по названию сервиса. Существующие записи **не перезаписываются**.

### ❓ Можно ли импортировать данные из других менеджеров паролей?

Прямой импорт из других менеджеров не поддерживается. Вы можете:
1. Экспортировать данные из другого менеджера в CSV/JSON
2. Преобразовать в формат PassGen (вручную или скриптом)
3. Импортировать через JSON

## Шифратор

### ❓ Что такое шифратор сообщений?

Шифратор позволяет зашифровать произвольный текстовый пароль с использованием алгоритма ChaCha20-Poly1305.

### ❓ Как зашифровать сообщение?

1. Перейдите на вкладку **Шифратор**
2. Выберите режим **"Зашифровать"**
3. Введите текст сообщения
4. Введите пароль для шифрования
5. Нажмите **"Зашифровать"**
6. Скопируйте результат

### ❓ Как дешифровать сообщение?

1. Перейдите на вкладку **Шифратор**
2. Выберите режим **"Дешифровать"**
3. Вставьте зашифрованный текст
4. Введите тот же пароль
5. Нажмите **"Дешифровать"**
6. Скопируйте расшифрованное сообщение

### ❓ Насколько безопасно шифрование?

Шифрование использует тот же алгоритм (ChaCha20-Poly1305), что и хранение паролей. Безопасность зависит от надёжности пароля шифрования.

### ❓ Можно ли восстановить сообщение без пароля?

**Нет.** Без пароля дешифрование невозможно. Это особенность алгоритма AEAD.

## Проблемы и решения

### ❌ Неверный PIN-код

**Проблема:** Приложение сообщает о неверном PIN-коде.

**Решение:**
- Проверьте раскладку клавиатуры
- Убедитесь, что не нажат Caps Lock
- После 5 неудачных попыток подождите 30 секунд

**Если забыли PIN:** Восстановление невозможно. Требуется переустановка приложения.

### ❌ Не могу скопировать пароль

**Проблема:** Кнопка копирования не работает.

**Решение:**
1. Проверьте разрешения приложения
2. Попробуйте скопировать пароль вручную (выделение → копирование)
3. Перезапустите приложение

### ❌ Данные не сохраняются

**Проблема:** Пароли не сохраняются в хранилище.

**Решение:**
1. Проверьте свободное место на устройстве
2. Убедитесь, что приложение имеет доступ к хранилищу
3. Проверьте логи безопасности
4. Перезапустите приложение

### ❌ Ошибка при импорте

**Проблема:** Не удаётся импортировать файл.

**Решение:**
1. Проверьте формат файла (JSON или .passgen)
2. Убедитесь, что файл не повреждён
3. Для .passgen проверьте мастер-пароль
4. Проверьте права доступа к файлу

### ❌ Приложение закрывается при неактивности

**Проблема:** Приложение автоматически блокируется.

**Решение:** Это нормальное поведение. Таймер неактивности: 5 минут. Введите PIN-код для разблокировки.

### ❌ Медленная работа приложения

**Проблема:** Приложение работает медленно.

**Решение:**
1. Закройте другие приложения
2. Проверьте свободное место
3. Перезапустите приложение
4. Если записей очень много (>1000), рассмотрите архивацию старых

### ❌ Ошибка шифрования/дешифрования

**Проблема:** Не удаётся зашифровать или дешифровать данные.

**Решение:**
1. Проверьте целостность данных
2. Убедитесь, что используется правильный пароль
3. Проверьте логи ошибок
4. При экспорте .passgen убедитесь в правильности мастер-пароля
