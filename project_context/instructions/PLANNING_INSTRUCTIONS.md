## 1. НАЗНАЧЕНИЕ

Эта инструкция описывает процесс создания, обновления и хранения планов разработки для проекта PassGen.

---

## 2. ТИПЫ ПЛАНОВ

### 2.1 Стратегический план (PROJECT_PLAN.md)

**Расположение:** `project_context/planning/PROJECT_PLAN.md`

**Назначение:** Долгосрочное планирование проекта (все этапы)

**Структура:**
```markdown
# 📊 План развития проекта PassGen

## 1. Текущее состояние
[Анализ текущего состояния]

## 2. Выполненные этапы
[Список завершённых этапов]

## 3. План развития (ROADMAP)
[Этапы 8, 9, 10...]

## 4. Следующие шаги
[Немедленные действия]

## 5. Заключение
[Выводы]
```

**Когда обновлять:**
- После завершения каждого этапа
- При изменении приоритетов
- Еженедельно (проверка актуальности)

---

### 2.2 План этапа (TASK_PLAN_N.md)

**Расположение:** `project_context/planning/TASK_PLAN_N.md`

**Назначение:** Детальное планирование конкретного этапа

**Структура:**
```markdown
# 📋 План Этапа N: [Название]

**Дата создания:** YYYY-MM-DD
**Версия:** 1.0
**Статус:** Черновик/В работе/Завершено

## 1. Цель этапа
[Описание цели]

## 2. Задачи
### 2.1 Критические
- [ ] Задача 1
- [ ] Задача 2

### 2.2 Важные
- [ ] Задача 3

### 2.3 Желательные
- [ ] Задача 4

## 3. Сроки
- Начало: YYYY-MM-DD
- Окончание: YYYY-MM-DD
- Дедлайн: YYYY-MM-DD

## 4. Ресурсы
- Файлы для создания
- Файлы для обновления
- Зависимости

## 5. Критерии успеха
[Как поймём что этап завершён]

## 6. Риски
[Возможные проблемы и решения]

## История изменений
[Таблица версий]
```

**Когда создавать:**
- Перед началом нового этапа
- При получении новой задачи

## 3. ПРОЦЕСС ПЛАНИРОВАНИЯ

### 3.1 Создание плана

**Шаг 1: Определение цели**
```
Вопросы:
- Что нужно достичь?
- Зачем это нужно?
- Как измерим результат?
```

**Шаг 2: Декомпозиция**
```
Разбить цель на задачи:
- Критические (обязательно)
- Важные (желательно)
- Желательные (по возможности)
```

**Шаг 3: Оценка**
```
Для каждой задачи:
- Время выполнения (часы)
- Необходимые ресурсы
- Зависимости от других задач
```

**Шаг 4: Приоритизация**
```
Метод MoSCoW:
- Must have (критично)
- Should have (важно)
- Could have (желательно)
- Won't have (отложить)
```

**Шаг 5: Документирование**
```
Создать файл плана по шаблону
```

---

### 3.2 Обновление плана

**Ежедневная проверка:**
```markdown
## Статус на YYYY-MM-DD

### Выполнено
- [x] Задача 1

### В работе
- [ ] Задача 2 (50%)

### Проблемы
- Проблема 1
- Решение: [описание]
```

**Еженедельное обновление:**
```markdown
## Неделя N

### Прогресс
[Диаграмма/GANTT]

### Отклонения
[Что пошло не по плану]

### Корректировки
[Изменения в плане]
```

---

### 3.3 Завершение плана

**Чек-лист завершения:**
```markdown
## Критерии завершения

- [ ] Все критические задачи выполнены
- [ ] Важные задачи выполнены на 80%+
- [ ] Создан отчёт об этапе
- [ ] Проведено ревью
- [ ] Обновлён PROJECT_PLAN.md
```

**Финальные действия:**
1. Установить статус `Завершено`
2. Создать `STAGE_N_COMPLETE.md`
3. Обновить `CURRENT_PROGRESS.md`
4. За коммитить изменения

### 5.1 Матрица Эйзенхауэра

```
┌─────────────────┬─────────────────┐
│   СРОЧНО        │   НЕ СРОЧНО     │
│   ВАЖНО         │   ВАЖНО         │
├─────────────────┼─────────────────┤
│ 🔴 Делать       │ 🟡 Планировать  │
│                 │                 │
└─────────────────┴─────────────────┘
┌─────────────────┬─────────────────┐
│   СРОЧНО        │   НЕ СРОЧНО     │
│   НЕ ВАЖНО      │   НЕ ВАЖНО      │
├─────────────────┼─────────────────┤
│ 🟡 Делегировать │ ❌ Удалить      │
│                 │                 │
└─────────────────┴─────────────────┘
```

### 5.2 Оценка сложности

| Оценка | Время | Описание |
|---|---|---|
| XS | < 1 часа | Очень маленькая |
| S | 1-4 часа | Маленькая |
| M | 4-8 часов | Средняя |
| L | 1-2 дня | Большая |
| XL | 2-5 дней | Очень большая |
| XXL | > 5 дней | Критически большая (разбить!) |

## 6. МОНИТОРИНГ ВЫПОЛНЕНИЯ

### 6.1 Диаграмма прогресса

```markdown
## Прогресс этапа

```
[████████░░] 80%

Задачи:
✅ 8/10 выполнены
🔄 1 в работе
⏳ 1 ожидает
```
```

### 6.2 Метрики

| Метрика | Формула | Цель |
|---|---|---|
| **Выполнено задач** | Completed / Total * 100% | 100% |
| **Точность оценки** | Estimated / Actual * 100% | 80-120% |
| **Скорость** | Tasks / Week | Рост |
| **Технический долг** | Debt Tasks / Total * 100% | < 20% |

## 8. ЧАСТЫЕ ОШИБКИ

### ❌ Ошибка 1: Слишком крупные задачи

**Проблема:**
```markdown
- [ ] Реализовать аутентификацию (40 часов)
```

**Решение:**
```markdown
- [ ] Создать AuthScreen (8 часов)
- [ ] Создать AuthController (8 часов)
- [ ] Реализовать PIN input (8 часов)
- [ ] Добавить блокировку (8 часов)
- [ ] Интегрировать логирование (8 часов)
```

---

### ❌ Ошибка 2: Нет критериев успеха

**Проблема:**
```markdown
- [ ] Улучшить производительность
```

**Решение:**
```markdown
- [ ] Снизить время загрузки с 2с до 500мс
- [ ] Уменьшить размер APK на 20%
```
