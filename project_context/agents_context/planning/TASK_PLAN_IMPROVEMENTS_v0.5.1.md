### 2.1 🔴 Увеличить итерации PBKDF2 до 100K+

**Проблема:** Текущее значение 10,000 итераций недостаточно для современной безопасности.

**Решение:**
```dart
// Было
static const int _pbkdf2Iterations = 10000;

// Стало
static const int _pbkdf2Iterations = 100000;
```

**Файлы:**
- `lib/core/utils/crypto_utils.dart`

**Критерии успеха:**
- [ ] Итерации увеличены до 100,000
- [ ] Миграция для существующих пользователей (ребаланс хеша)
- [ ] Тесты пройдены
- [ ] Производительность в норме (<1 сек на аутентификацию)

---

### 2.2 🔴 Включить логирование в транзакцию

**Проблема:** Логирование событий безопасности происходит вне транзакции, что может привести к несогласованности данных.

**Решение:**
```dart
await database.transaction(() async {
  // Основная операция
  await db.insert('password_entries', entry);
  
  // Логирование внутри транзакции
  await db.insert('security_logs', {
    'action_type': actionType,
    'timestamp': timestamp,
    'details': details,
  });
});
```

**Файлы:**
- `lib/data/repositories/password_entry_repository_impl.dart`
- `lib/data/repositories/security_log_repository_impl.dart`

**Критерии успеха:**
- [ ] Все операции с PWD_* логгируются внутри транзакции
- [ ] Тесты на транзакционность
- [ ] Откат при ошибке включает логи

### 2.3 🟡 Добавить проверку дубликатов

**Проблема:** Пользователь может сохранить несколько паролей для одного сервиса.

**Решение:**
```dart
// Проверка перед вставкой
final existing = await db.query(
  'password_entries',
  where: 'service = ? AND login = ?',
  whereArgs: [service, login],
);

if (existing.isNotEmpty) {
  throw DuplicateEntryException('Запись уже существует');
}
```

**Файлы:**
- `lib/data/repositories/password_entry_repository_impl.dart`
- `lib/domain/entities/password_entry.dart` (новый exception)

**Критерии успеха:**
- [ ] Проверка на дубликаты (service + login)
- [ ] Пользовательское уведомление
- [ ] Опция "Обновить существующую запись"

---

### 2.4 🟡 Добавить обработку ошибок с rollback

**Проблема:** При ошибках в середине транзакции данные могут остаться в частичном состоянии.

**Решение:**
```dart
try {
  await database.transaction(() async {
    await db.insert('password_entries', entry);
    await db.insert('password_configs', config);
    await db.insert('security_logs', log);
  });
} catch (e) {
  // Transaction автоматически откатывается
  _logger.severe('Transaction failed: $e');
  rethrow;
}
```

**Файлы:**
- `lib/data/repositories/*_repository_impl.dart`

**Критерии успеха:**
- [ ] try-catch во всех транзакциях
- [ ] Логирование ошибок
- [ ] Тесты на откат транзакций
