## 1. ОБЛАСТЬ ПРИМЕНЕНИЯ

Настоящая политика безопасности распространяется на все компоненты приложения PassGen:
- Аутентификация и авторизация
- Шифрование данных
- Хранение конфиденциальной информации
- Логирование событий безопасности
- Управление сессиями

## 2. СТАНДАРТЫ КРИПТОГРАФИИ

### 2.1 Алгоритмы шифрования

| Алгоритм | Назначение | Параметры | Стандарт |
|---|---|---|---|
| **ChaCha20-Poly1305** | Шифрование данных | AEAD, 256-bit ключ | ✅ RFC 8439 |
| **PBKDF2-HMAC-SHA256** | Деривация ключей | 10,000 итераций | ✅ NIST SP 800-132 |
| **CSPRNG** | Генерация случайных чисел | `Random.secure()` | ✅ NIST SP 800-90A |

### 2.2 Требования к ключам

| Параметр | Требование | Текущее значение | Статус |
|---|---|---|---|
| Длина ключа шифрования | ≥256 бит | 256 бит | ✅ |
| Длина nonce | ≥96 бит | 256 бит (32 байта) | ✅ |
| Длина MAC (Poly1305) | 128 бит | 128 бит (16 байт) | ✅ |
| Итерации PBKDF2 | ≥10,000 | 10,000 | ✅ |
| Длина соли PBKDF2 | ≥128 бит | 256 бит (32 байта) | ✅ |

### 2.3 Запрещённые алгоритмы

❌ **ЗАПРЕЩЕНО:**
- MD5, SHA1 (хэширование)
- DES, 3DES, RC4 (шифрование)
- ECB режим (блочные шифры)
- Предсказуемые генераторы случайных чисел (`Random()` без `.secure()`)

## 3. ПОЛИТИКА АУТЕНТИФИКАЦИИ

### 3.1 PIN-код

| Требование | Значение | Обоснование |
|---|---|---|
| Минимальная длина | 4 цифры | Базовая защита |
| Максимальная длина | 8 цифр | Удобство использования |
| Формат | Только цифры (0-9) | Мобильная клавиатура |
| Хранение | PBKDF2 хэш с солью | Защита от перебора |

### 3.2 Защита от подбора

| Параметр | Значение | Действие |
|---|---|---|
| Максимум попыток | 5 | Блокировка |
| Длительность блокировки | 30 секунд | Задержка атаки |
| Сброс попыток | После успешного входа | Нормализация |

### 3.3 Автоблокировка сессии

| Событие | Таймаут | Действие |
|---|---|---|
| Неактивность пользователя | 5 минут | Возврат к AuthScreen |
| Сворачивание приложения | Мгновенно | Блокировка |
| Переключение приложения | Мгновенно | Блокировка |

## 4. ПОЛИТИКА ХРАНЕНИЯ ДАННЫХ

### 4.1 Классификация данных

| Класс | Примеры | Защита |
|---|---|---|
| **Критические** | Мастер-ключ, PIN-хэш | Только RAM, затирание |
| **Конфиденциальные** | Пароли, конфиги | Шифрование ChaCha20 |
| **Внутренние** | Настройки, логи | SQLite без шифрования |
| **Публичные** | Категории, иконки | SQLite без ограничений |

### 4.2 Хранение ключей

| Ключ | Где хранится | Время жизни |
|---|---|---|
| **Мастер-ключ** | Оперативная память | До выхода из сессии |
| **Ключ шифрования** | Деривируется из мастер-пароля | До завершения операции |
| **Соль PBKDF2** | SQLite (app_settings) | Постоянно |
| **Nonce** | Вместе с зашифрованными данными | Постоянно |

### 4.3 Затирание чувствительных данных

**Обязательно затирать:**
- [ ] Мастер-ключ после выхода из сессии
- [ ] Ключи шифрования после использования
- [ ] Пароли в буфере обмена (60 сек)
- [ ] Временные буферы с чувствительными данными

**Метод затирания:**
```dart
// Заполнение нулями
for (int i = 0; i < sensitiveData.length; i++) {
  sensitiveData[i] = 0;
}
```

## 5. ПОЛИТИКА ЛОГИРОВАНИЯ

### 5.1 Типы событий безопасности

| Событие | Код | Данные |
|---|---|---|
| Успешная аутентификация | `AUTH_SUCCESS` | Timestamp |
| Неудачная аутентификация | `AUTH_FAILURE` | Timestamp, попытка |
| Блокировка | `AUTH_LOCKOUT` | Timestamp, причина |
| Установка PIN | `PIN_SETUP` | Timestamp |
| Смена PIN | `PIN_CHANGED` | Timestamp |
| Удаление PIN | `PIN_REMOVED` | Timestamp |
| Создание пароля | `PWD_CREATED` | Timestamp, service |
| Просмотр пароля | `PWD_ACCESSED` | Timestamp, service |
| Обновление пароля | `PWD_UPDATED` | Timestamp, service |
| Удаление пароля | `PWD_DELETED` | Timestamp, service |
| Экспорт данных | `DATA_EXPORT` | Timestamp, формат |
| Импорт данных | `DATA_IMPORT` | Timestamp, формат |
| Изменение настроек | `SETTINGS_CHG` | Timestamp, настройка |

### 5.2 Запрет на логирование

❌ **КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО ЛОГИРОВАТЬ:**
- Пароли в любом виде
- Мастер-ключи
- PIN-коды
- Полные данные банковских карт
- Секретные ключи шифрования

### 5.3 Хранение логов

| Параметр | Требование |
|---|---|
| Максимальный размер | 10,000 записей |
| Срок хранения | 90 дней |
| Шифрование | Не требуется (нет чувствительных данных) |
| Доступ | Только через UI приложения |

## 6. ПОЛИТИКА ЭКСПОРТА/ИМПОРТА

### 6.1 Поддерживаемые форматы

| Формат | Расширение | Шифрование | Статус |
|---|---|---|---|
| **PassGen** | `.passgen` | ChaCha20-Poly1305 | ✅ Обязательно |
| **JSON Mini** | `.json` | ChaCha20-Poly1305 | ✅ Обязательно |
| **CSV** | `.csv` | Нет | 🔲 Перспектива |

### 6.2 Требования к экспорту

1. **Обязательное шифрование** для всех форматов
2. **Уникальный nonce** для каждого файла
3. **Проверка MAC** при импорте
4. **Валидация заголовка** формата

### 6.3 Структура файла .passgen

```
┌─────────────────────────────────────┐
│ HEADER: "PASSGEN_V1" (10 байт)      │
├─────────────────────────────────────┤
│ VERSION: 1 (1 байт)                 │
├─────────────────────────────────────┤
│ FLAGS: 0 (1 байт)                   │
├─────────────────────────────────────┤
│ NONCE: 32 байта                     │
├─────────────────────────────────────┤
│ DATA_LENGTH: 4 байта                │
├─────────────────────────────────────┤
│ DATA: зашифрованный JSON            │
├─────────────────────────────────────┤
│ MAC: 16 байт (Poly1305 tag)         │
└─────────────────────────────────────┘
```

## 7. ПОЛИТИКА БЕЗОПАСНОСТИ КОДА

### 7.1 Запрещённые паттерны

❌ **ЗАПРЕЩЕНО:**
```dart
// Print в production
print('Password: $password');

// Хардкод ключей
const key = 'my_secret_key';

// Логирование чувствительных данных
log('User entered: $pin');

// Предсказуемые случайные числа
final random = Random(); // ❌ без .secure()
```

### 7.2 Обязательные проверки

✅ **ОБЯЗАТЕЛЬНО:**
```dart
// Валидация входных данных
if (!isValidPinFormat(pin)) {
  throw ValidationFailure();
}

// Проверка MAC
if (!macVerified) {
  throw EncryptionFailure();
}

// Обработка ошибок
try {
  await encrypt(data);
} catch (e) {
  throw EncryptionFailure();
}
```

### 7.3 Code Review требования

Перед merge в main:
- [ ] Нет `print()` в production коде
- [ ] Нет хардкода ключей/паролей
- [ ] Все чувствительные данные зашифрованы
- [ ] Логи не содержат секретов
- [ ] Используется `Random.secure()`

## 8. УПРАВЛЕНИЕ УЯЗВИМОСТЯМИ

### 8.1 Классификация уязвимостей

| Уровень | Описание | Срок исправления |
|---|---|---|
| 🔴 **Критический** | Утечка ключей, обход аутентификации | 24 часа |

1. **Обнаружение** → Создание issue
2. **Оценка** → Классификация уязвимости
3. **Исправление** → Fix в отдельной ветке
4. **Проверка** → Security review
5. **Релиз** → Hotfix версия
