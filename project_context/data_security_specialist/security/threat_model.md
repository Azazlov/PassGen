## 1. ОБЗОР

Этот документ описывает модель угроз безопасности для приложения PassGen — кроссплатформенного менеджера паролей с локальным шифрованием.

---

## 2. ОБЪЕКТ МОДЕЛИРОВАНИЯ

### 2.1 Описание системы

**Приложение:** PassGen  
**Тип:** Кроссплатформенный менеджер паролей  
**Платформы:** Windows, Linux, Android  
**Архитектура:** Clean Architecture (Flutter/Dart)  
**Хранение данных:** Локальное (SQLite)  
**Шифрование:** ChaCha20-Poly1305

## 4. УГРОЗЫ

### 4.1 Классификация угроз (STRIDE)

| Категория | Описание | Примеры |
|---|---|---|
| **S**poofing | Выдача себя за другого | Подбор PIN |
| **T**ampering | Изменение данных | Модификация БД |
| **R**epudiation | Отрицание действий | Удаление логов |
| **I**nformation Disclosure | Утечка информации | Кража паролей |
| **D**enial of Service | Отказ в обслуживании | Блокировка приложения |
| **E**levation of Privilege | Повышение привилегий | Обход аутентификации |

## 7. СЦЕНАРИИ АТАК

### 7.1 Сценарий 1: Физический доступ к устройству

**Атакующий:** Злоумышленник с физическим доступом

**Цель:** Получение доступа к паролям

**Шаги атаки:**
1. Получение доступа к устройству
2. Попытка подбора PIN (5 попыток)
3. ⛔ **Блокировка на 30 секунд**
4. Попытка дампа памяти
5. ⚠️ **Ключи могут остаться в RAM**
6. Кража файла БД
7. ⛔ **Данные зашифрованы ChaCha20**

**Результат:** ⚠️ Частичный успех (требуется дополнительная защита RAM)

---

### 7.2 Сценарий 2: Вредоносное приложение

**Атакующий:** Вредоносное приложение на устройстве

**Цель:** Перехват паролей

**Шаги атаки:**
1. Мониторинг буфера обмена
2. ⚠️ **Успех в течение 60 секунд**
3. Попытка чтения файлов приложения
4. ⛔ **Изоляция песочницы**
5. Перехват файла экспорта
6. ⛔ **Файл зашифрован**

**Результат:** ⚠️ Частичный успех (буфер обмена)

---

### 7.3 Сценарий 3: Офлайн-атака на БД

**Атакующий:** Злоумышленник с копией БД

**Цель:** Расшифровка паролей

**Шаги атаки:**
1. Копирование файла БД
2. Анализ структуры SQLite
3. ⛔ **Пароли зашифрованы**
4. Попытка подбора мастер-ключа
5. ⛔ **PBKDF2 10,000 итераций**
6. Атака на ChaCha20
7. ⛔ **256-bit ключ, уникальные nonce**

**Результат:** ❌ Провал (криптографически стойкая защита)

## 8. ДОПУЩЕНИЯ И ОГРАНИЧЕНИЯ

### 8.1 Допущения

| Допущение | Обоснование |
|---|---|
| ОС не скомпрометирована | Root/jailbreak вне модели |
| Пользователь не злоумышленник | Социальная инженерия вне модели |
| Устройство не потеряно | Физическая защита устройства |
| Нет network-атак | Приложение не использует сеть |

### 8.2 Ограничения модели

- ❌ Не рассматриваются атаки на уровне ОС
- ❌ Не рассматривается социальная инженерия
- ❌ Не рассматривается hardware-атаки (JTAG, Chip-off)
- ❌ Не рассматриваются side-channel атаки по времени
